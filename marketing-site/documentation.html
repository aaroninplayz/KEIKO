Abstract & Platform Overview

Recruitment has traditionally relied on human judgment to evaluate a candidate's technical expertise, communication abilities, confidence, and overall suitability for a role. While experienced interviewers can often identify talented candidates, the process itself remains inherently subjective. Every interviewer has a unique style, evaluates candidates differently, and unconsciously prioritizes different aspects of performance. Two candidates providing identical answers may receive entirely different evaluations simply because of interviewer perception, fatigue, bias, or environmental conditions.

Modern recruitment presents additional challenges. Organizations often receive hundreds or even thousands of applications for a single position, making it increasingly difficult to provide every candidate with a consistent, fair, and comprehensive evaluation. Recruiters are expected to simultaneously observe technical accuracy, body language, eye contact, speech clarity, confidence, professionalism, and behavioral characteristics while maintaining a natural conversation. Human cognitive limitations make it nearly impossible to accurately monitor all these dimensions simultaneously.

The Philosophy Behind KEIKO

The primary design philosophy behind KEIKO is based on one simple principle: An interview should measure more than just the correctness of answers.

A candidate's success during an interview depends on multiple independent factors working together, including technical knowledge, problem-solving ability, communication effectiveness, confidence, behavioral consistency, professional presentation, eye contact, posture, speech clarity, adaptability, learning ability, and context retention. Traditional systems evaluate only the verbal responses provided. KEIKO transforms these subjective observations into measurable telemetry by continuously collecting structured metrics throughout the interview lifecycle.

The Problem KEIKO Solves

Subjective Evaluation

Human interviewers naturally possess unconscious biases (accent, speaking speed, appearance). KEIKO minimizes these inconsistencies by generating objective, measurable metrics.

Limited Human Attention

Recruiters cannot track eye contact, body language, tone, and technical correctness simultaneously. KEIKO distributes these tasks among independent AI agents collaborating in real time.

Lack of Constructive Feedback

Candidates receive simple pass/fail decisions. KEIKO generates detailed, diagnostic dashboards breaking down category-wise performance and areas for improvement.

Generic Questions

Instead of static question banks, KEIKO tailors follow-ups dynamically by understanding the uploaded resume, projects, and target job description requirements.

Multi-Agent AI Advantage

Rather than employing a single, monolithic LLM (which would face severe latency and context limitations if asked to simultaneously process video, audio, generate questions, and track state), KEIKO distributes responsibilities across specialized AI agents. This guarantees lower latency, better fault isolation, modular scaling, and provider flexibility.

Objectives & Scope

KEIKO was conceived to address the absence of an objective, repeatable, and data-driven method for evaluating candidate performance. Instead of functioning as a static questionnaire, it behaves as an adaptive interviewer modifying its strategy based on candidate depth and composure metrics.

Primary Functional Objectives

1. Intelligent Candidate & Job Understanding

Performs semantic analysis of uploaded resumes (education, experience, technical stack, projects) and Job Descriptions (skills clusters, experience requirements, seniority) to tailor the conversation context.

2. Real-Time Behavioral & Adaptive Intelligence

Continuous longitudinal monitoring of visual signals (eye contact, posture alignment, expressions) and voice attributes (WPM speed, verbal hesitations, clarity). Adjusts question difficulty dynamically based on answer coherence.

3. Quantifiable Feedback

Converts qualitative behaviors into explicit, normalized scores. Generates comprehensive reports tracking performance timelines and actionable roadmaps instead of a simple pass/fail flag.

Platform Limitations

  • Visual tracking coordinates degrade in poor/unbalanced lighting environments or low frame-rate webcams.
  • Vocal pacing and pause detections depend on clean microphone inputs and low ambient room noise.
  • WebSocket real-time telemetry requires stable network connectivity to prevent packet buffering lag.

Platform Core Principles

KEIKO is built on four core guidelines: Objectivity (quantifiable evidence), Personalization (session uniqueness), Transparency (traceable scores), and Provider Independence (decoupled LLM vendor APIs to switch models seamlessly).

Technology Stack & Infrastructure

The KEIKO platform is designed as a modern python web application. The backend coordinates core evaluation tasks asynchronously while the frontend renders cockpit metrics dynamically:

Backend Engine — FastAPI & Python

Uses FastAPI's ASGI architecture for asynchronous routing. Coordinates static profiles matching, WebSocket sessions, database queries, and background thread execution.

Core Endpoint: ./api/v1/interview/ws/{session_id}
Databases — SQLite

Maintains two databases: app.db (persists session records, metadata, configurations, and reports) and test_realtime.db (used for telemetric testing and streaming logs). Abstracted repository interfaces allow seamless migration to enterprise SQL/NoSQL platforms.

Computer Vision & Speech Stack — MediaPipe & OpenCV

Integrates OpenCV for video frame processing. Real-time visual tracking uses MediaPipe Mesh models to calculate landmarks (iris ratios, head orientations, and shoulder posture vectors). Speech streams are converted to transcripts and analyzed for filler words.

UI Rendering — Tailwind & JavaScript

Uses Tailwind CSS utilities and vanilla JavaScript to manage raw WebSocket payloads, update the DOM without heavy frame re-renders, and plot SVG charts.

System Architecture

The architecture of an Artificial Intelligence platform determines its scalability, maintainability, fault tolerance, and overall performance. Since KEIKO simultaneously processes live video streams, microphone audio, resume documents, job descriptions, AI-generated conversations, and real-time behavioral analytics, a conventional monolithic application architecture would quickly become inefficient and difficult to maintain.

To address these challenges, KEIKO adopts a layered modular architecture in which each subsystem performs a single well-defined responsibility while communicating through structured interfaces. Rather than allowing every component to directly interact with every other module, the architecture organizes functionality into logical layers that exchange only the information required to complete a specific task. This separation of concerns significantly reduces system complexity, improves code maintainability, and allows independent scaling of computationally intensive components such as Computer Vision, Speech Processing, and Large Language Models.

Five Execution Layers

Layer 1 — Input Layer

The Input Layer serves as the entry point into the KEIKO platform. Every interview session begins here. It handles resume and job description uploads (PDF/DOCX), parses user preferences, and manages websocket handshake configurations.

  • Supported Formats: PDF, DOCX (with future markdown/HTML support)
  • Session Creation: Unique Session ID (e.g. UUID 4d93b7fa-c71d-41f6-b5c7-3d5d6dfb44ea)
  • Normalizes user-defined slider metrics prior to database storage
Layer 2 — Intelligence Layer

The Intelligence Layer forms the computational core of KEIKO. It consists of multiple specialized AI agents operating independently and concurrently:

  • Resume Intelligence: Extracts certifications, timelines, work experience, and skills vectors.
  • JD Intelligence: Maps required technologies, responsibilities, and seniority expectations.
  • Video Intelligence: Integrates posture_analyzer.py, eye_contact_analyzer.py, and facial_expression_analyzer.py to track facial landmarks and frame alignment.
  • Audio Intelligence: Implements voice_analyzer.py to track speaking speed (WPM), hesitation pause events, and filler count.
  • LLM Intelligence: Connects to provider model boundaries to resolve interview queries.
Layer 3 — Interview Layer

The Interview Layer coordinates active sessions and tracks conversation flow. Implemented inside orchestrator.py, router.py, and conversation_manager.py, it maintains context and difficulty adjustments:

  • Conversation State: Remembers dialogue history (Question ➔ Response ➔ Follow-up) to prevent repetitive questions.
  • Adaptive Difficulty: Adjusts complexity dynamically based on candidate performance metrics.
Layer 4 — Evaluation Layer

Once telemetry has been collected from all modules, the Evaluation Layer transforms raw observations into performance scores. Using validation and normalization pipelines configured via weight_config.py, it computes category ratings (Technical Competency, Communication, Posture, Eye Contact, Confidence) using normalized weighted sums.

Layer 5 — Reporting Layer

The Reporting Layer converts numerical evaluations into structured visual summaries. Rather than presenting raw statistics, the reporting engine generates SVG radar charts, timeline graphs, strength/weakness tables, and personalized coaching roadmaps.

Session Manager

Encapsulates sessions dynamically inside isolated states. Persists profile info, telemetry histories, question records, and final evaluations inside SQLite databases (app.db, test_realtime.db) to support simultaneous concurrent runs.

Thread Architecture

Spawns concurrent worker threads (Main Server, Resume parsing subthread, Video parsing subthread, Audio parsing subthread, Evaluation engine subthread) to isolate processor-heavy OpenCV/MediaPipe calculations.

Fault Tolerance & Resilience

KEIKO targets modular, graceful degradation to ensure session stability:

  • If the webcam disconnects, audio analysis continues.
  • If speech transcription temporarily fails, video analysis remains active.
  • If one AI provider becomes unavailable, fallback models can be hot-swapped dynamically.
  • Temporary network packet dropouts trigger automatic client reconnection attempts.

Scalability Strategy

While the platform operates as a unified application, its decoupled, modular design naturally supports future migration to independent microservices (Resume Service, Video Analysis Service, Audio Analysis Service, Evaluation Service, Reporting Service) connected through API gateways or message queues.

PROVIDER AGNOSTIC ORCHESTRATION

AI Model Architecture

Artificial Intelligence is the core driving force behind KEIKO. However, unlike many AI-powered applications that tightly couple their business logic to a single Large Language Model (LLM) provider, KEIKO follows a provider-agnostic architecture. This design philosophy ensures that the interview engine, evaluation framework, and user experience remain independent of the underlying AI model responsible for generating interview questions and understanding candidate responses.

The rapid evolution of Generative AI has resulted in numerous capable models such as OpenAI GPT-4o, Google Gemini, Anthropic Claude, Meta Llama, Mistral, DeepSeek, and locally hosted models through Ollama. Each model differs in pricing, latency, reasoning capabilities, privacy guarantees, context window size, and deployment flexibility. Rather than forcing organizations to adopt one ecosystem, KEIKO abstracts all model interactions behind a unified orchestration layer. The Interview Intelligence Engine communicates only with this abstraction layer, while the orchestration layer handles provider-specific API formats, authentication, prompt construction, response parsing, retry logic, and error handling.

The Orchestration Pipeline

Implemented inside conversation_manager.py and routed via the WebSocket interfaces, the orchestration layer performs prompt construction, token tracking, and response parsing:

Interview Request ➔ Prompt Builder ➔ Context Manager ➔ Provider Selector ➔ Model Adapter ➔ Response Parser
Prompt Builder

Dynamically builds prompts by combining the Candidate Resume Profile (skills, stack), Job Description targets, previously asked questions, current difficulty levels, and conversational composure metrics.

Context Manager

Tracks the history of conversation exchanges (Questions & Responses) in sequential context windows, prompting the AI to maintain continuous context instead of generating repetitive, unrelated queries.

Model Adapter Layer

Converts unified internal payloads into specific vendor schemas (e.g. converting standardized system-message arrays to OpenAI, Gemini, or Claude formats).

Response Parser

Normalizes varying provider formats into structured JSON variables, validating question string parameters and topic tag categories before returning them to the Interview Engine.

Redundant Provider Failover

Maintains request retry policies. If the primary LLM provider times out or triggers rate-limiting parameters, the orchestration loop dynamically switches routes to fallback models.

Local Inference & Data Privacy

To address data residency and privacy rules, KEIKO supports local-first model inference. By configuring the adapter to target local ports (e.g. running Llama-3 locally via Ollama), organizations can execute visual-vocal training sessions completely offline without sending data to external APIs.

Phase 1 — Session Setup

The candidate configures the session parameters, uploading their resume (PDF/DOCX) and Job Description (PDF/TXT) files, selecting standard difficulty levels (Easy, Medium, Hard, Expert), and setting scoring weights prioritizations.

Calibration: weight_config.py normalizes inputs to sum to exactly 1.0 (100%)
Phase 2 — Workspace Initialization

Initiated by `POST /api/v1/session/initialize`, the server prepares the session workspace to minimize latency during streaming:

  • Generates unique Session UUIDs (e.g. a4ef72cd-918d-4a5f-bc2f-5e82a4d8138d).
  • Inserts initial session rows and weights configs into the SQLite database records (app.db).
  • Triggers background tasks in the parser threads to index resume experiences and extract Job Description expectations.
Phase 3 — WebSocket Connection

Establishes a persistent, bi-directional WebSocket connection via the relative path ./api/v1/interview/ws/{session_id}.

The server accepts the connection and returns a JSON connected handshake event verifying connection readiness, while concurrently spinning up heartbeat monitors to track network packet drops.

Phase 4 — Real-Time Streaming

The browser webcam captures frames, encodes them as Base64 JPEG strings, and streams them over the socket at a rate of 10 FPS. Simultaneously, the mic streams binary PCM audio buffers.

  • Camera: Frame Capture ➔ JPEG ➔ Base64 String ➔ WebSocket
  • Microphone: PCM Audio Chunks ➔ Audio Buffer ➔ WebSocket
  • Server: Distributes visual frames and audio chunks to parallel processing threads
Phase 5 — Adaptive Coaching & Dialogue

Coordinated via conversation_manager.py. After transcription, the Dialogue Engine analyzes candidate answers against semantic job profiles, dynamically generating targeted technical follow-up questions.

  • Adaptive Pacing: Adjusts subsequent question complexity (Easy ➔ Architect ➔ Scenarios) based on answer depth.
  • Continuous Telemetry: Spawns subthreads to track posture deviations and gaze offset vectors over time.
Phase 6 — Evaluation & Report Generation

Upon connection close, live telemetry streaming stops. The Evaluation Engine consolidates all recorded logs, applies normalized weight coefficients to compute category scores (Technical, Communication, Professionalism, Pacing), and triggers the Reporting Engine to generate SVG radar charts and personalized coaching roadmaps.

Dual Deployment Model (Live Gateway + Local Docker)

To optimize deployment and keep hosting costs minimal, KEIKO implements a segregated hosting design:

  • Static Portal Gateway (24/7 Live Website): Hosts documentation, tutorials, user profiles, and static pages on the public web server.
  • Telemetry Sandbox Container (Dockerized Engine): The heavy-duty media analyzer is packaged as a local Dockerfile package. Candidates pull the container locally to host the CPU/GPU-bound MediaPipe/OpenCV pipelines and WebSocket endpoint on their own machine, preserving absolute privacy and preventing external bandwidth costs.

COMPUTER VISION SENSORS

Video Intelligence Module

In modern interviews, communication extends far beyond verbal responses. Human interviewers subconsciously evaluate a wide range of non-verbal behavioral cues including eye contact, posture, facial expressions, attentiveness, confidence, and overall professional presence. These behavioral indicators often influence hiring decisions as much as technical knowledge, yet they are inherently subjective and susceptible to personal bias.

KEIKO addresses this challenge through its Video Intelligence Module, a real-time computer vision subsystem responsible for continuously monitoring candidate behavior during the interview. Rather than recording video solely for playback, the module processes incoming webcam frames in real time to extract measurable behavioral metrics that contribute to the overall interview evaluation.

Visual Processing Flow

Webcam Capture ➔ Downsample (10 FPS) ➔ Base64 Encode ➔ WebSocket Tunnel ➔ MediaPipe Mesh Pipeline

Face Mesh & Landmarks Mapping

Using MediaPipe's Face Mesh models, the CV pipeline extracts structural coordinate arrays representing anatomical landmarks (pupil positions, eyelids, nose contours, jawline):

Gaze Contact Scoring

Implemented in eye_contact_analyzer.py. Calculates pupil iris centers relative to eye contours to measure gaze alignment towards screen focus. Looking away for more than 2.0 seconds registers focus warnings.

Head Pose Orientation

Estimates Head Pitch (looking up/down), Yaw (looking left/right), and Roll (side tilts) using landmark geometry to monitor distraction behaviors over time.

Expressions Classification

Calculated inside facial_expression_analyzer.py. Analyzes landmark changes to classify neutral, happy, focused, confused, or nervous states.

Posture Analysis & Scoring

Implemented inside posture_analyzer.py, the posture estimator checks the geometric angle between the candidate's nose, shoulders, and vertical axis:

Angle Range Rating Scoring Impact
160° – 180° Excellent / Ideal No penalty, standard scores preserved
145° – 159° Good / Slight Slouch Minor baseline warnings recorded
130° – 144° Warning / Slouching Active slouch warnings triggered, small score penalty
< 130° Poor Alignment Deductions applied to Professional Presence baseline

VOCAL & SPEECH SENSORS

Audio Intelligence Module

Effective communication is one of the most important aspects of a successful interview. While technical knowledge determines whether a candidate possesses the required expertise, the ability to communicate ideas clearly, confidently, and professionally often determines how that expertise is perceived by interviewers. Factors such as speech clarity, speaking pace, pronunciation, hesitation, confidence, and response structure collectively influence interview outcomes.

Traditional interview platforms primarily record audio for later playback, requiring manual review by recruiters or candidates. This approach is time-consuming and often subjective, as different evaluators may interpret the same response differently. KEIKO overcomes this limitation through its Audio Intelligence Module, an independent real-time speech analysis subsystem responsible for continuously monitoring and evaluating the candidate's verbal communication throughout the interview.

Auditory Processing Flow

Microphone ➔ Raw PCM Buffering ➔ Bandpass Noise Filter ➔ STT Transcription Engine ➔ Pace & Tone Telemetry
PCM Stream Buffering

Mic inputs are sliced into 3-5 second binary PCM chunks (16-bit, mono channel, sampled at 16kHz) to minimize streaming network overhead and reduce transcript processing latency.

Speech-to-Text (STT)

Converts speech audio inputs into clean text strings, forwarding the transcript directly to the Interview Dialogue Engine to grade question responses.

Speaking Pace & Words-Per-Minute (WPM)

Calculated inside voice_analyzer.py, words-per-minute (WPM) metrics are computed continuously by dividing the word count of transcribed sentences by the speaking duration:

Speaking Speed Pacing Classification Comprehension Quality
< 110 WPM Too Slow / Deliberate May indicate difficulty recalling info or uncertainty
130 WPM – 150 WPM Optimal Speed Clear cadence, highly professional, easy to follow
151 WPM – 170 WPM Moderate Fast Acceptable pacing but can degrade in complex system topics
> 170 WPM Too Fast / Rushed Likely reduces clarity, signals high nervousness levels
Hesitation & Filler Detections

Analyzes spoken text against verbal hesitation lists to extract filler tags (e.g. "um", "uh", "like", "you know", "basically", "actually") to score vocal presence.

Vocal Modulation & Pitch

Monitors tone fluctuations and rhythm deviations, helping distinguish structured verbal delivery from monotonous or flat acoustic responses.

ANALYTIC GRADING CONSOLE

Evaluation Framework

The ultimate objective of an interview is not merely to ask questions but to accurately evaluate a candidate's suitability for a role. Traditional interview evaluations are often subjective, relying heavily on the personal judgment of interviewers. Factors such as interviewer experience, unconscious bias, memory limitations, and inconsistent evaluation criteria can significantly influence hiring decisions. Two interviewers assessing the same candidate may produce entirely different conclusions despite observing identical responses.

KEIKO addresses this challenge through a comprehensive Evaluation Framework that transforms interview observations into objective, measurable, and explainable performance metrics. Rather than relying on a single overall score, the framework evaluates multiple dimensions of candidate performance, including technical competency, communication effectiveness, behavioral presence, confidence, and engagement.

Evaluation Pipeline Flow

Telemetry Capture ➔ Metric Extraction ➔ Score Calculation ➔ Weight Normalization (weight_config.py) ➔ Reporting Engine

Multi-Dimensional Assessments

Technical Competency

Meases candidate subject-matter depth, accuracy, and engineering design thinking. This category holds the largest baseline scoring ratio.

Communication Skills

Tracks vocal pacing clarity (words-per-minute target loops) and structural organization of responses.

Professional Presence

Monitors eye contact stability (gaze alignment vectors) and shoulder posture posture calculations.

Confidence Indicators

Monitors verbal hesitation patterns, pause intervals, and expressiveness to track confidence levels over time.

METRIC COEFFICIENT CALIBRATOR

Weights & Normalizer API

Different organizations evaluate candidates using different hiring priorities. A software startup may emphasize technical expertise above all else, while a customer-facing organization may prioritize communication skills and professional presence. Likewise, senior engineering roles often require deeper architectural reasoning, whereas entry-level positions may focus more on learning ability and problem-solving potential.

Recognizing that no single evaluation model is universally applicable, KEIKO incorporates a Weights & Normalization API, a configurable subsystem that enables interview administrators or candidates to define the relative importance of different evaluation parameters before an interview session begins. Instead of relying on fixed scoring formulas, the platform allows individual evaluation metrics to be assigned configurable weight coefficients, automatically normalized so that the total contribution equals 1.0 (100%).

Session-Level Safeguards & Validation Rules

To maintain evaluation integrity, weight configurations are stored per session inside app.db and validated against server-side validation rules (missing parameters checks, empty config checks, negative value restrictions). Weight profiles become completely immutable after the session WebSocket connection officially establishes to prevent rating manipulation.

REAL-TIME COMMUNICATIONS SOCKET

WebSocket Telemetry Packets

Modern AI-powered interview systems require continuous communication between the client application and the backend server. Unlike conventional web applications that rely on individual HTTP requests for every interaction, KEIKO operates as a real-time intelligent system where video frames, microphone audio, interview questions, behavioral telemetry, and evaluation metrics must flow continuously throughout the interview session.

To achieve this, KEIKO employs WebSocket-based bidirectional communication, allowing the client and server to exchange information instantly without repeatedly establishing new network connections. This persistent communication channel forms the backbone of the platform, enabling synchronized multimedia streaming, adaptive AI interactions, and real-time telemetry generation.

To maintain connection state, client and server exchange cyclic ping / pong frames. In the event of network dropout, the client-side socket registers a connection warning and immediately triggers auto-reconnection scripts to verify the UUID session and resume the interview thread.

The interview process does not end when the final question is answered. In many traditional interview environments, candidates receive little or no feedback regarding their performance. Recruiters may internally evaluate responses, but candidates often remain unaware of their strengths, weaknesses, or areas requiring improvement. This lack of actionable feedback significantly reduces the educational value of mock interviews.

KEIKO addresses this limitation through its Performance Reporting & Candidate Feedback Dashboard, a comprehensive reporting subsystem that consolidates technical evaluations, behavioral telemetry, communication analysis, and interview intelligence into a unified performance report. Rather than providing only a single interview score, KEIKO presents detailed multidimensional analytics that help candidates understand exactly how they performed and identify practical strategies for improvement.

Knowledge Base & System FAQ

Frequently Asked Questions

Comprehensive intelligence regarding KEIKO architecture, telemetry scoring, deployment options, and data privacy protocols.

dataset General Intelligence

What is KEIKO?

KEIKO is an advanced multi-agent AI platform for talent intelligence and candidate self-training. It processes multi-modal sensor inputs (vision, audio, natural language) in real-time to generate objective, engineering-grade interview evaluation telemetry.

How does KEIKO differ from traditional interview tools?

Traditional tools assess only the textual correctness of spoken or written answers. KEIKO evaluates full-spectrum interview performance including body language, gaze contact, vocal pacing, filler frequency, and structured technical reasoning.

memory Technology Stack & Execution

What infrastructure powers the real-time AI agents?

KEIKO runs on a FastAPI backend with asynchronous WebSocket channels, MediaPipe visual posture tracking, WebAudio API processing, and integrated LLM evaluation chains running locally or via API endpoints.

Can KEIKO operate locally without external cloud connectivity?

Yes. KEIKO supports a local application node configuration where backend servers and lightweight models execute on localhost (port 8000), eliminating external API token dependencies.

shield Data Privacy & Security

How is candidate video and audio telemetry handled?

All sensory video frames and audio chunks are processed transiently in memory. Numerical telemetry metrics (gaze coordinates, posture angles, WPM) are sanitized before saving to local SQLite stores.
